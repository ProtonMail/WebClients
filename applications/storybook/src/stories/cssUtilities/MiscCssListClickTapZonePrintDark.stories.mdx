## Something that should appear on <code>hover</code> in a parent

This version below make an element with `className="opacity-on-hover"` appearing when hovering its container or when focus is inside its container (container needs `className="opacity-on-hover-container"`).

<div class="flex flex-justify-space-between mb2 on-mobile-flex-column">
  <div class="w49 flex-align-self-center relative">
    <div class="opacity-on-hover-container" tabIndex="0">
      hover or focus me, pikaaaaa…
      <button type="button" class="button button-outline-weak opacity-on-hover">CHU</button>
      </div>
      <br /><br />
    Careful when using this, is appropriated for non-critical actions, and for actions related to the container (not to hide everything). If you want to use it on an element that needs to be present on mobile/touch or for a critical action, you might use  `opacity-on-hover-only-desktop` helper (this one will make it always visible when user does not have a precise pointing capability).
    <br /><br />
    As `opacity-on-hover` relies on `visibility: hidden` (so we keep space), you can use `opacity-on-hover-no-width` if you want the element NOT taking space while hidden.
    <br /><br />
  </div>
  <div class="w49">
<Source
  language='html'
  light
  format
  code={`
<div class="opacity-on-hover-container">
  hover or focus inside me, pikaaaaa…
  <button className="opacity-on-hover">CHU</button>
</div>
`}
/>

  </div>
</div>

## light Mode

For React applications, a <code>islightMode</code> (<code>appConfigBody-lightmode</code> for Angular) class is applied to the <code>body</code> when light mode is triggered.

Then you have two helpers: <code>hide-on-lightmode</code> will hide something when light mode is triggered, and <code>display-on-lightmode</code> (which is hidden by default) will be displayed when light mode is triggered.
