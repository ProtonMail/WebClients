# Responsive utilities

The most common utilities can be applied conditionally at different breakpoints, making it easy to build complex responsive directly using classes on your components.

Here are those breakpoints:

To apply a utility selectively at a specific breakpoint, simply prepend the breakpoint name, followed by the `:` character, to the utility:

```html
<Component className="pl-4 sm:pl-8 md:pl-10 lg:pl-12 xl:pl-14">…</Component>
```

## This is mobile first

We've reversed the logic of responsive compared to legacy mechanics.

From now on, our responsive utilities use a **mobile-first approach**, similar to what you may have seen in other frameworks such as Bootstrap or Tailwind.

This means that as soon as you apply a utility class to an element, it affects all screen sizes, while **a responsive utility will affect screens of the desired size and above**.

Following this logic, if you want to apply a class only on mobile, for example, you'll have to override it with another utility for larger screen sizes. So if, for example, you want to apply a `flex` only on mobile, you'll need to apply a class to override it on larger screens, probably `block`:

```html
<Component className="flex sm:block">…</Component>
```
